# soul.md

> YouTube transcripts as structured, searchable, git-trackable Markdown.

## origin

Needed a way to capture spoken knowledge from YouTube — talks, tutorials, interviews — and make it *textual*. Searchable. Taggable. Compatible with Obsidian, Hugo, Jekyll, and anything that reads Markdown with YAML frontmatter.

CLI-first because that's where the work happens.

## becoming

The canonical transcript pipeline. Single video, search bundles, playlist imports, batch downloads. Already has a web UI for interactive use. The tool is mature — future growth is in enrichment and integration, not core features.
